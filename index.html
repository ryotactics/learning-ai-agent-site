AIエージェント 知識マップ

Claude Code / Codex を活用するために必要な大枠の概念を整理したリファレンス

1. AIエージェントとは

LLM + Memory + Planning + Tools = AI Agent

AIエージェントは「目標を理解し→計画を立て→行動し→結果を評価し→完了まで繰り返す」自律的なシステム。 単なるチャットボットと異なり、ツールを呼び出して現実世界に作用する。

2. 4 つのコアコンポーネント

🧠

LLM（推論エンジン）

入力を解釈し、次に取るべき行動を決定する中枢。Claude / GPT-5 などのモデルが担う。

💾

Memory（記憶）

In-context（会話履歴）・External（DB）・Episodic（過去の経験）の3層。なければ毎回ゼロスタート。

📋

Planning（計画）

複雑なタスクを小タスクへ分解し、実行順序を決める。ReAct・CoT・Task Decomposition が代表的手法。

🔧

Tools（ツール）

コード実行・ファイル操作・Web検索・API呼び出しなど、外部世界と接続する「手」。

5. Claude Code vs Codex 比較

Claude Code（Anthropic）
  • 形態： CLI / IDE 拡張 / Web
  • 実行環境： ローカル環境で直接動作
  • コンテキスト： コードベース全体を読む
  • マルチエージェント： サブエージェントを並列起動可
  • 拡張： MCP（Model Context Protocol）でツール追加
  • SDK： Agent SDK でカスタムエージェント構築可
Codex（OpenAI）
  • 形態： ChatGPT 統合 / CLI
  • 実行環境： クラウド上の隔離コンテナ
  • コンテキスト： GitHub リポジトリを接続
  • セキュリティ： タスク実行中はインターネット遮断
  • 特化： コーディングタスクに最適化
  • モデル： GPT-5.3-Codex（最新）

6. 押さえておきたい重要概念

用語意味関連ツール
Agentic Loop 目標達成まで Perceive→Reason→Act→Observe を繰り返すサイクル Claude Code, Codex
Tool Use / Function Calling LLM が構造化された形式で外部関数を呼び出す仕組み 全エージェント基盤
MCP（Model Context Protocol） AIツールと外部データソースをつなぐオープン標準プロトコル Claude Code
Multi-Agent 複数エージェントが協調して並列にタスクを処理するアーキテクチャ Claude Code, Managed Agents
RAG（Retrieval Augmented Generation） 外部DBから関連情報を取得してLLMの回答精度を高める手法 Memory ストア連携
ReAct Reasoning（思考）と Action（行動）を交互に行うプロンプト戦略 Planning 実装パターン
Guardrails エージェントの行動範囲を制限する安全装置（権限・スコープ設定） Managed Agents, Codex
Prompt Caching 同一コンテキストのトークンをキャッシュしてコスト・速度を改善する機能 Claude API
